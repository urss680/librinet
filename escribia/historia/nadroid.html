Para crear una aplicación de lectura de PDF paso a paso en Android Studio, la forma más sencilla es utilizar una biblioteca externa, ya que implementar un renderizador de PDF desde cero es complejo. Una de las bibliotecas más populares y fáciles de usar esAndroidPdfViewerde Barteksc.

Aquí tienes un tutorial paso a paso para crear una aplicación básica de lectura de PDF en Android Studio:

Paso 1: Crear un nuevo proyecto en Android Studio

  1. Abre Android Studio.
  2. Selecciona "New Project" (Nuevo Proyecto).
  3. Elige la plantilla "Empty Activity" (Actividad Vacía) y haz clic en "Next" (Siguiente).
  4. Configura tu proyecto:
    • Name (Nombre):LectorPDFApp(o el nombre que prefieras).
    • Package name (Nombre del paquete): Puedes dejar el predeterminado o cambiarlo.
    • Save location (Ubicación de guardado): Elige dónde guardar tu proyecto.
    • Language (Lenguaje):JavaoKotlin(para este tutorial usaremos Java, pero la lógica es similar).
    • Minimum SDK version (Versión mínima del SDK): Puedes elegir una versión común como API 21 o superior, ya quePdfRenderer(la base de muchas librerías) está disponible desde la API 21.
  5. Haz clic en "Finish" (Finalizar).

Paso 2: Añadir la dependencia de la biblioteca AndroidPdfViewer

  1. Abre el archivobuild.gradle(Module: app) de tu proyecto.
  2. Dentro de la seccióndependencies { }, añade la siguiente línea para la bibliotecaAndroidPdfViewer:gradle implementation 'com.github.barteksc:android-pdf-viewer:3.2.0-beta.1' Nota: La versión3.2.0-beta.1es una de las mencionadas en los resultados. Puedes buscar la última versión estable en el repositorio de GitHub de la biblioteca.
  3. Si usas versiones más recientes de Gradle o Android Studio, es posible que también necesites añadirjcenter()a tus repositorios en el archivosettings.gradle.ktsobuild.gradlede nivel de proyecto, yandroid.enableJetifier=trueengradle.propertiessi tienes problemas de compatibilidad.
  4. Sincroniza tu proyecto haciendo clic en "Sync Now" (Sincronizar ahora) en la barra de notificaciones de Android Studio.

Paso 3: Añadir permisos en AndroidManifest.xml (si es necesario) Si planeas abrir archivos PDF desde el almacenamiento externo del dispositivo, necesitarás permisos de lectura. Para PDFs en la carpetaassetsno es estrictamente necesario.

  1. Abre el archivoAndroidManifest.xml.
  2. Dentro de la etiqueta<manifest>, pero fuera de la etiqueta<application>, añade el siguiente permiso:xml <uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
  3. Para asignar más memoria a la aplicación, lo cual puede ser útil para PDFs grandes, puedes añadirandroid:largeHeap="true"a la etiqueta<application>:xml <application ... android:largeHeap="true"> ... </application>

Paso 4: Preparar un archivo PDF de ejemplo en la carpeta 'assets' Para empezar, cargaremos un PDF desde los recursos internos de la aplicación.

  1. En la ventana "Project" (Proyecto) de Android Studio (normalmente a la izquierda), haz clic derecho en la carpetaapp.
  2. SeleccionaNew>Folder>Assets Folder(Nueva > Carpeta > Carpeta de Assets).
  3. Haz clic en "Finish" (Finalizar).
  4. Copia un archivo PDF (por ejemplo,sample.pdf) y pégalo dentro de la carpetaassetsque acabas de crear (app/src/main/assets).

Paso 5: Diseñar el layout de la actividad (activity_main.xml)

  1. Abre el archivoactivity_main.xml(ubicado enapp/src/main/res/layout).

  2. Elimina elTextViewpredeterminado si lo hay.

  3. Añade elPDFViewde la biblioteca al layout. Puedes usar unLinearLayoutoConstraintLayoutcomo padre.

    &lt;com.github.barteksc.pdfviewer.PDFView
    android:id="@+id/pdfView"
    android:layout_width="match_parent"
    android:layout_height="match_parent" /&gt;
    </RelativeLayout>

    Nota: Se usóRelativeLayoutcomo ejemplo, pero puedes usarLinearLayoutconandroid:orientation="vertical"como se sugiere en.

Paso 7: Ejecutar la aplicación

  1. Conecta tu dispositivo Android a tu computadora o inicia un emulador.
  2. Haz clic en el botón "Run" (Reproducir) (el icono de triángulo verde) en la barra de herramientas de Android Studio.

Tu aplicación debería iniciarse y mostrar el archivo PDF que colocaste en la carpetaassets.

Funcionalidades adicionales y consideraciones:

  • Abrir PDF desde almacenamiento local: Para abrir un PDF desde el almacenamiento del dispositivo, necesitarás un selector de archivos. Una vez que tengas elUrioFiledel PDF, puedes usarpdfView.fromUri(uri).load()opdfView.fromFile(file).load().
  • Abrir PDF desde una URL: La biblioteca también permite cargar PDFs directamente desde una URL usandopdfView.fromUrl("url_del_pdf").load().
  • Manejo de errores y carga: Para una aplicación más robusta, puedes añadir callbacks a la función.load()para saber cuándo el PDF se ha cargado correctamente o si ha habido un error.
  • Navegación entre páginas: La bibliotecaAndroidPdfViewerofrece funciones básicas como zoom, desplazamiento y navegación por páginas.
  • Permisos en tiempo de ejecución: Para Android 6.0 (API 23) y versiones posteriores, el permisoREAD_EXTERNAL_STORAGEdebe solicitarse en tiempo de ejecución si el PDF no está en la carpetaassets.
  • Alternativas aAndroidPdfViewer:
    • PdfRenderer (API de Android): Disponible desde la API 21, permite una renderización básica de PDFs sin librerías externas pesadas. Es más ligero pero requiere más código para implementar funcionalidades avanzadas.
    • Librerías comerciales: Existen opciones como Nutrient o ComPDFKit que ofrecen características más avanzadas (anotaciones, edición, firmas digitales) pero son de pago.
    • WebView: Para casos muy básicos, podrías intentar mostrar PDFs incrustados en Google Docs Viewer a través de un WebView, pero esto requiere una conexión a internet y el PDF debe ser accesible públicamente.

Este enfoque utilizando la bibliotecaAndroidPdfVieweres uno de los métodos más directos para integrar una funcionalidad de lectura de PDF en tu aplicación Android.
